# Label Studio 本地开发搭建手册

## 1. 文档目标

这份文档用于帮助团队在当前仓库基础上快速搭建本地开发环境，重点解决：

- 后端如何启动
- 前端如何启动
- 前后端如何联调
- Windows / PowerShell 下如何执行
- 当前仓库里有哪些“文档提到但实际缺失”的前置文件

本文基于当前仓库实际内容编写，尽量只写“当前代码仓里可以直接执行”的路径。

---

## 2. 开发环境建议

## 2.1 推荐工具版本

结合当前仓库配置，建议本地环境至少满足：

- Python：`3.10` 到 `3.13`
- Node.js：建议 `22.x`
- Yarn：使用仓库锁文件安装
- Docker Desktop：最新版
- Git：最新版

说明：

- `pyproject.toml` 要求 Python `>=3.10,<4`
- `Dockerfile` 中默认 `NODE_VERSION=22`、`PYTHON_VERSION=3.13`
- 前端依赖较重，Node 版本过旧时更容易出现构建问题

## 2.2 Windows 环境说明

当前工作环境是 Windows + PowerShell，建议：

- Python、Node、Docker 都加入 PATH
- PowerShell 用管理员权限启动一次，确认脚本权限正常
- 遇到 `lxml` 安装失败时，优先升级 `pip`，必要时按官方文档补装 wheel

## 3. 仓库初始化

## 3.1 克隆并进入项目

```powershell
git clone <your-repo-url> Label-Studio
cd E:\Label-Studio
```

如果仓库已经存在，直接进入根目录即可。

## 3.2 先确认关键文件存在

根目录应至少包含：

- `pyproject.toml`
- `poetry.lock`
- `web/package.json`
- `Makefile`
- `docker-compose.yml`
- `Dockerfile`

如果这些文件缺失，不建议继续按本文执行。

---

## 4. 后端本地启动

## 4.1 安装 Poetry

```powershell
pip install poetry
```

验证：

```powershell
poetry --version
```

## 4.2 安装后端依赖

在仓库根目录执行：

```powershell
poetry install
```

说明：

- 这一步会安装 Django、DRF、数据库驱动、数据处理依赖等
- 首次安装时间可能较长

## 4.3 迁移数据库

当前仓库最简单的本地开发模式是 SQLite。

执行：

```powershell
make migrate-dev
```

如果本机没有 `make`，可以直接执行等价命令：

```powershell
$env:DJANGO_DB="sqlite"
$env:LOG_DIR="tmp"
$env:DEBUG="true"
$env:LOG_LEVEL="DEBUG"
$env:DJANGO_SETTINGS_MODULE="core.settings.label_studio"
poetry run python label_studio/manage.py migrate
```

## 4.4 启动 Django 开发服务

推荐命令：

```powershell
make run-dev
```

如果本机没有 `make`，执行等价命令：

```powershell
$env:DJANGO_DB="sqlite"
$env:LOG_DIR="tmp"
$env:DEBUG="true"
$env:LOG_LEVEL="DEBUG"
$env:DJANGO_SETTINGS_MODULE="core.settings.label_studio"
poetry run python label_studio/manage.py runserver
```

默认访问地址：

- `http://localhost:8080`

说明：

- `label_studio/manage.py` 已把 Django 默认端口设为 `8080`
- 本地开发模式下通常不需要先配 PostgreSQL

## 5. 前端本地启动

## 5.1 安装前端依赖

进入 `web` 目录：

```powershell
cd web
yarn install --frozen-lockfile
```

## 5.2 最简单启动方式

如果你只是想让前端进入开发监听模式，可执行：

```powershell
yarn dev
```

或者在根目录执行：

```powershell
make frontend-dev
```

说明：

- 默认会按 `web/README.md` 的 HMR 逻辑运行
- 前端热更新服务默认地址通常为 `http://localhost:8010`

## 5.3 推荐联调方式

推荐开两个终端：

### 终端 1：后端

```powershell
cd E:\Label-Studio
make run-dev
```

### 终端 2：前端

```powershell
cd E:\Label-Studio\web
yarn dev
```

这样启动后：

- Django 负责 API 和页面宿主
- 前端 HMR 负责热更新静态资源

## 6. HMR 配置

## 6.1 当前仓库现状

`web/README.md` 中说明可以在根目录创建 `.env` 启用 HMR。

当前仓库里：

- 存在 `docker-compose.override.example.yml`
- 不存在 `.env.development`

这意味着：

- `Makefile` 里的 `env-dev-setup` 会尝试复制 `.env.development`
- 但当前仓库并没有这个文件
- 因此不要直接依赖 `make docker-dev-setup`

## 6.2 建议手动创建 `.env`

在仓库根目录新建 `.env`，内容可以先写：

```env
FRONTEND_HMR=true
FRONTEND_HOSTNAME=http://localhost:8010
DJANGO_HOSTNAME=http://localhost:8080
```

说明：

- `FRONTEND_HMR=true` 用于通知 Django 使用前端热更新资源
- `FRONTEND_HOSTNAME` 默认为 `http://localhost:8010`
- `DJANGO_HOSTNAME` 默认为 `http://localhost:8080`

如果不需要 HMR，这个文件可以先不配。

## 8. 推荐的本地开发流程

## 8.1 只改后端

适合：

- API 开发
- 数据模型改造
- 导入导出逻辑改造

流程：

1. `poetry install`
2. `make migrate-dev`
3. `make run-dev`
4. 浏览器访问 `http://localhost:8080`

## 8.2 前后端联调

适合：

- 页面开发
- 表单/接口联调
- 设置页改造

流程：

1. 根目录创建 `.env`
2. 启动 Django
3. 启动 `web` 前端 dev 服务
4. 通过浏览器访问 `http://localhost:8080`

## 8.3 编辑器功能开发

适合：

- 标注交互改造
- 新标签能力开发

流程：

1. 先把基础前后端跑通
2. 进入 `web`
3. 使用 `yarn lsf:serve` 或相关 watch 命令
4. 结合 `web/apps/playground` 或主应用验证效果

## 9. 常见问题

## 9.1 `make` 命令不可用

Windows 上经常没有 `make`。

处理方式：

- 使用 Git Bash / WSL 安装 `make`
- 或直接执行 Makefile 中对应的原始命令

本文已经给出了后端核心命令的 PowerShell 等价写法。

## 9.2 `poetry install` 失败

优先检查：

- Python 版本是否满足 `>=3.10`
- `pip` 是否过旧
- 网络是否能访问依赖源

建议：

```powershell
python -m pip install --upgrade pip
pip install poetry
```

## 9.3 Windows 下 `lxml` 安装报错

这是 Label Studio 文档中明确提到的常见问题。

建议顺序：

1. 先升级 `pip`
2. 重新安装
3. 必要时按官方建议安装 wheel 包

## 9.4 前端启动后页面没热更新

优先检查：

- 根目录是否有 `.env`
- `.env` 中是否设置了 `FRONTEND_HMR=true`
- 后端是否重启过
- `yarn dev` 是否真正启动成功

## 9.5 端口冲突

默认涉及这些端口：

- `8080`
  - Django 本地开发服务
- `8010`
  - 前端 HMR 服务
- `8000`
  - 容器内 uWSGI/Nginx upstream 使用

如果 `8080` 被占用，需要先释放端口或调整本地启动方式。

---

## 10. 本地联调完成的最小验收标准

建议至少达到以下状态，才算本地开发环境搭好：

1. `poetry install` 成功
2. `yarn install --frozen-lockfile` 成功
3. Django 能在 `http://localhost:8080` 正常启动
4. 前端 dev 服务能正常启动
5. 浏览器能打开主页面
6. 修改一个前端文件后能看到热更新或重新构建结果

---

## 11. 推荐下一步

本地环境跑通之后，建议下一步做下面其中一项：

1. 跑通 `docker compose up`，熟悉生产默认部署方式
2. 新增一个最小业务页面，验证前后端联调链路
3. 深入阅读：
   - `label_studio/core/urls.py`
   - `label_studio/projects`
   - `web/apps/labelstudio`
   - `web/libs/editor`

如果团队后续要正式部署，继续看根目录的 `DEPLOYMENT_GUIDE.md`。

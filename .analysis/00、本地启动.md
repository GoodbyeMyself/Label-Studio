## 启动顺序

  1. 启动后端 Django
     目录：E:\Label-Studio
     命令：

  .\.venv\Scripts\python.exe label_studio\manage.py runserver 0.0.0.0:8080

  2. 启动前端 HMR
     目录：E:\Label-Studio\web
     命令：

  $env:NODE_ENV="development"
  $env:BUILD_NO_SERVER="true"
  yarn ls:dev

## 访问方式

  只访问：

  http://localhost:8080

  不要直接拿 http://localhost:8010 当系统入口，也不要和 127.0.0.1 混用。


  8080 是 Django 服务，是真正的 Label Studio Web 入口。你访问系统、登录、拿后端页面模板、调后端 API，本质都走它。
  8010 是前端 HMR 开发服务，只负责实时编译并提供前端 JS/CSS 资源，用来热更新。

  在你这个项目里，正常开发模式是这样配合的：

  - 浏览器访问 http://localhost:8080
  - Django 返回页面 HTML
  - 这个 HTML 里会把前端静态资源指到 http://localhost:8010/react-app/...
  - 所以前端代码改动后，8010 会立刻重新编译，页面热更新
  - 但登录态、接口、模板、用户 session，还是以 8080 为准

  所以你可以把它理解成：

  - 8080 = 主站 / 后端 / 登录和 API
  - 8010 = 前端开发资源服务器 / 热更新

  你会看到的结果

  - 8080：系统页面、登录、接口
  - 8010：前端热更新资源服务
  - 改前端代码后，在 8080 页面里就能看到效果

 ## 建议固定配置

  根目录 /E:/Label-Studio/.env 里保留这几项：

  FRONTEND_HMR=true
  FRONTEND_HOSTNAME=http://localhost:8010
  DJANGO_HOSTNAME=http://localhost:8080
  CSRF_TRUSTED_ORIGINS=http://localhost:8010,http://127.0.0.1:8010,http://localhost:8080,http://127.0.0.1:8080

 ## 关闭时

  各自终端里按 Ctrl + C 即可。
